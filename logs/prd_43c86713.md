# 개인 맞춤형 RSS 뉴스 큐레이터 with AI 요약

## 1. 한 줄 정의
관심 RSS 피드를 등록하면 AI가 자동으로 기사를 요약하고 키워드 기반으로 필터링해주는 개인용 뉴스 큐레이션 웹 서비스

## 2. 문제 정의
매일 수십 개의 RSS 피드에서 쏟아지는 기사들을 일일이 확인하는 것은 시간 낭비이며, 정작 중요한 정보를 놓치기 쉽습니다. 기존 RSS 리더는 단순 나열만 제공하고, 유료 큐레이션 서비스는 비용 부담이 큽니다. 본인이 관심 있는 키워드 중심으로 자동 요약·우선순위화된 뉴스를 매일 받아볼 수 있다면, 정보 과부하 없이 효율적인 뉴스 소비가 가능합니다.

## 3. 타겟 사용자
- **1차**: 개발자 본인 (기술 뉴스, 특정 관심사 추적용)
- **2차**: 정보 과부하에 시달리는 지식 노동자, 연구자, 블로거
- **특성**: RSS 피드를 활용하며, 매일 10개 이상의 소스를 확인해야 하는 사람

## 4. 핵심 기능
1. **RSS 피드 등록 및 자동 수집**: 사용자가 URL 입력 시 feedparser로 주기적 크롤링 (매 6시간)
2. **AI 3줄 요약**: 로컬 LLM(Ollama + Llama 3.2) 또는 GPT-4o-mini로 기사당 150토큰 이내 요약 생성
3. **키워드 필터링 및 우선순위**: 등록 키워드 매칭 시 별표 표시 및 상단 노출
4. **카테고리 자동 분류**: AI가 기술/경제/사회 등 5개 카테고리로 자동 분류
5. **일일 다이제스트 대시보드**: React로 구현된 타임라인 뷰 + 읽음/북마크 관리

## 5. 기술 스택
- **백엔드**: Python (Flask/FastAPI), feedparser, APScheduler, SQLite
- **AI**: Ollama (Llama 3.2 3B) 로컬 운영 또는 OpenAI GPT-4o-mini (일 50개 제한)
- **프론트엔드**: React (Vite), Axios, TailwindCSS
- **배포**: Railway/Render (백엔드), Vercel (프론트엔드), SQLite 파일 기반 DB
- **기타**: tenacity (API 재시도), python-dotenv

## 6. 예상 일정
- **1-2주**: 환경 설정, RSS 파싱 + 저장 로직 구현 (feedparser, SQLite 스키마)
- **3-4주**: 로컬 LLM 연동 및 요약 파이프라인 구축 (Ollama 설치, 프롬프트 튜닝)
- **5주**: 키워드 필터링 + 카테고리 분류 로직 구현
- **6주**: Flask API 엔드포인트 완성 (CRUD, 스케줄러 통합)
- **7주**: React 대시보드 UI/UX 구현 (기사 리스트, 필터, 북마크)
- **8주**: 배포, 테스트, 문서화 (README, API 명세)
