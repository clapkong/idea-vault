# 내 플레이리스트 분석기 (My Playlist Analyzer)

## 1. 한 줄 정의
Spotify 플레이리스트를 분석해서 내 음악 취향을 시각화하고 재미있는 통계를 보여주는 파이썬 프로그램

## 2. 문제 정의
음악을 좋아하지만 내가 어떤 장르를 주로 듣는지, 어떤 특징의 음악을 선호하는지 정확히 모르는 경우가 많습니다. Spotify는 이런 데이터를 제공하지만 깊이 있는 분석이나 맞춤형 시각화는 제공하지 않습니다. 이 프로젝트는 자신의 음악 취향을 객관적으로 파악하고, 친구들과 공유할 수 있는 재미있는 통계를 제공합니다.

## 3. 타겟 사용자
- Spotify를 사용하는 음악 애호가
- 자신의 음악 취향을 분석하고 싶은 사람
- 친구들과 음악 취향을 비교하고 싶은 사람
- (본인) 개발 입문자로서 실제 사용할 수 있는 첫 프로젝트를 만들고 싶은 사람

## 4. 핵심 기능
1. **Spotify 플레이리스트 불러오기**: Spotify API를 통해 자신의 플레이리스트 데이터 가져오기
2. **장르 분석**: 플레이리스트에서 가장 많이 듣는 장르 TOP 5를 막대 그래프로 표시
3. **음악 특성 분석**: 에너지, 댄스성, 템포 등의 평균값을 계산하고 레이더 차트로 시각화
4. **재미있는 통계**: "가장 신나는 곡", "가장 우울한 곡", "총 재생 시간" 등의 정보 출력
5. **결과 저장**: 분석 결과를 텍스트 파일과 이미지로 저장해서 공유 가능

## 5. 기술 스택
- **언어**: Python 3.8+
- **필수 라이브러리**:
  - `spotipy` (Spotify API 사용)
  - `matplotlib` (그래프 그리기)
  - `pandas` (데이터 정리)
- **인증**: Spotify OAuth 2.0 (무료 계정으로 가능)
- **개발 도구**: VS Code 또는 PyCharm Community Edition

## 6. 예상 일정
- **1주차**:
  - 1-2일: Spotify Developer 계정 생성, API 키 발급, `spotipy` 설치 및 인증 연습
  - 3-4일: 플레이리스트 데이터 불러오기 코드 작성, 데이터 구조 파악
  - 5-7일: 장르 분석 기능 구현, 기본 통계 계산 코드 작성
  
- **2주차**:
  - 8-10일: `matplotlib`으로 그래프 그리기 (막대 그래프, 레이더 차트)
  - 11-12일: 재미있는 통계 추가, 결과 저장 기능 구현
  - 13-14일: 테스트, 버그 수정, 코드 정리, README 작성

## 7. 주요 리스크
1. **OAuth 인증 어려움**
   - 대응: Spotify Developer 공식 문서와 `spotipy` 튜토리얼 먼저 따라하기. 인증만 별도로 1일 할애
   
2. **그래프 라이브러리 학습 시간 부족**
   - 대응: `matplotlib` 기본 예제 코드 복사해서 데이터만 바꿔서 사용. 완벽한 디자인보다 작동 우선
   
3. **2주 일정이 빠듯함**
   - 대응: 핵심 기능(1, 2, 4)만 1주차에 완성. 시각화(3, 5)는 여유 있으면 추가하는 선택 기능으로 변경 가능
   
4. **API 사용량 제한**
   - 대응: Spotify 무료 플랜은 충분한 할당량 제공. 테스트 시 작은 플레이리스트로 시작
